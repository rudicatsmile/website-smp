# Rules for Antigravity Agent

## CHANGELOG Policy
- **Update CHANGELOG.md Automatically:** Every time a new feature is successfully implemented, or a bug is successfully fixed, you MUST automatically record the changes in the `CHANGELOG.md` file located at the root of the project.
- Do not wait for the user to instruct you to update the changelog. Make it a standard part of your completion process.
- Use the standard `[Unreleased]` format: group under `### Added`, `### Changed`, `### Fixed`, or `### Removed`.

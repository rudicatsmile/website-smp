    B. Notifikasi WhatsApp/Email Otomatis
        :: Notifikasi otomatis ke orang tua untuk: ketidakhadiran anak, pengumuman penting, jadwal pengambilan rapor, 
            tagihan jatuh tempo.

        Manual override (Filament)
            - Aksi baris **Siswa › Kirim Notifikasi** (pilih kanal + isi pesan custom)
            - Aksi bulk **Pembayaran › Kirim Pengingat** (pilih beberapa tagihan → kirim semua)
            - Aksi header **Log Notifikasi › Kirim Tes** (tes ke no HP/email tertentu)
            - Aksi baris **Log Notifikasi › Kirim Ulang** (retry notif yang gagal)

        Catatan
            - Format nomor HP otomatis dinormalisasi (`08xx` / `8xx` → `628xx`)
            - Field `parent_email` ditambahkan ke siswa (opsional)
            - Untuk testing tanpa gateway: set `WABLAS_DRIVER=log` (pesan WA hanya dicatat ke Laravel log)


        Konfigurasi .env

            # WhatsApp (Wablas)
            WABLAS_BASE_URL=https://console.wablas.com/api
            WABLAS_TOKEN=your_wablas_token
            WABLAS_SECRET=your_wablas_secret   # opsional tergantung paket
            WABLAS_DRIVER=http                 # 'log' untuk dev (tidak kirim nyata)
            WABLAS_DELAY_MS=800                # jeda antar pesan

            # Feature toggles (default true)
            NOTIF_ABSENSI_ENABLED=true
            NOTIF_PAYMENT_ENABLED=true
            NOTIF_ANNOUNCEMENT_ENABLED=true

            # Identitas sekolah untuk template
            NOTIF_SCHOOL_NAME="SMP Al Wathoniyah 9"
            NOTIF_SCHOOL_PHONE=
            NOTIF_SCHOOL_WEBSITE=

            # Queue (default pakai queue.default, biasanya database)
            NOTIF_QUEUE_NAME=notifications

    C. Surat Izin Online ✅ (sudah diimplementasi)

    D. Chatbot / FAQ Interaktif
        :: Bot FAQ berbasis keyword matching untuk menjawab pertanyaan umum (PPDB, biaya, fasilitas) 24/7. 
            Tersedia sebagai floating widget di semua halaman publik dan halaman dedicated `/faq`.


        Cara pakai
            1. Floating widget — 
                tombol hijau di pojok kanan bawah semua halaman publik. Klik untuk membuka panel chat, ketik pertanyaan, bot akan mencari FAQ yang cocok.
            2. Halaman FAQ di `/faq` — 
                menampilkan semua FAQ yang dikelompokkan per kategori (PPDB, Biaya, Fasilitas, Akademik, Umum). Bisa difilter per kategori. Panel chat juga tersedia di sidebar kiri.
            3. Admin Filament di menu FAQ — 
                CRUD FAQ: tambah/edit/hapus pertanyaan dan jawaban, atur kategori, kata kunci, urutan tampilan, dan status aktif/nonaktif.

        Cara kerja bot
            - Mencocokkan pertanyaan user dengan kata kunci (`keywords`) setiap FAQ
            - Jika cocok 1 FAQ: langsung tampilkan jawaban
            - Jika cocok beberapa: tampilkan daftar pertanyaan terkait
            - Jika tidak cocok: tampilkan pesan fallback + tawarkan kontak WhatsApp
            - Feedback thumbs up/down dicatat untuk analytics

        Log percakapan
            - Tersimpan di tabel `chat_logs` (session_id, pertanyaan, jawaban, faq_id, helpful)
            - Bisa digunakan untuk analisis FAQ mana yang sering ditanyakan / belum terjawab

    E. Buku Penghubung Digital ✅ (sudah diimplementasi)
        :: Komunikasi 2 arah berbasis topik antara orang tua ↔ wali kelas terkait perkembangan
            anak (akademik, perilaku, kehadiran, kesehatan). Setiap topik punya thread chat sendiri
            dengan lampiran, status (terbuka/dibalas/selesai/ditutup), dan notifikasi WA + Email
            otomatis setiap ada balasan.

        Cara pakai
            1. Orang Tua → `/portal/ortu` (ortu@smpalwathoniyah9.sch.id / password)
                - Login → di kartu anak, klik tombol **Buku Penghubung** (badge merah = pesan
                  belum dibaca).
                - Klik **Topik Baru** → isi subjek, kategori, prioritas, pesan + lampiran (opsional).
                - Topik akan terkirim dengan kode `BP-XXXXXX`; wali kelas otomatis dapat notif
                  WA + Email.
                - Lanjutkan chat di halaman detail topik. Klik **Tandai Selesai** jika sudah cukup.
            2. Wali Kelas / Admin → Filament `/admin` menu **Komunikasi › Buku Penghubung**
                - Wali kelas hanya melihat topik dari kelas yang dia walikan; admin & super_admin
                  melihat semua.
                - Buka topik → tab **Thread Pesan** → action **Balas Orang Tua**
                  (toggle Catatan Internal untuk catatan privat antar guru, tidak dikirim ke ortu).
                - Wali kelas juga bisa **inisiasi topik baru** lewat tombol "New" di list (mis.
                  melaporkan perilaku siswa ke orang tua) — pesan awal di form akan tersimpan
                  sebagai pesan pertama otomatis.
                - Status berubah: open → replied saat wali balas → open lagi saat ortu balas →
                  resolved/closed saat ditandai selesai. Badge angka kuning di sidebar menampilkan
                  jumlah topik aktif.

        Notifikasi otomatis
            - Saat orang tua kirim/balas → notif ke wali kelas (WA via `staff_members.whatsapp/phone`
              + Email via `staff_members.email`).
            - Saat wali kelas balas → notif ke orang tua (WA & Email via `students.parent_phone` &
              `students.parent_email`).
            - **Internal note (`is_internal=true`) tidak memicu notif** dan tidak terlihat oleh
              orang tua di portal.

        Konfigurasi .env
            NOTIF_PARENT_NOTE=true

        Tabel database
            - `parent_notes` (topik): code, student_id, school_class_id, homeroom_teacher_id,
              initiator_user_id, initiator_type, subject, category, priority, status, timestamps
            - `parent_note_messages` (thread): parent_note_id, sender_type (parent/teacher/system),
              user_id, staff_member_id, body, attachments (json), is_internal, read_at, timestamps

    F. Manajemen Materi Pelajaran Harian ✅ (sudah diimplementasi)
        :: Perencanaan kurikulum per kelas + pelaksanaan harian oleh guru. Kepala Kurikulum
            menyusun template kurikulum (daftar topik per pekan), lalu bulk-apply ke rentang
            tanggal jadi sesi mengajar konkret. Guru mengeksekusi lewat halaman "Mengajar
            Hari Ini" dan mencatat realisasi (pencapaian, catatan, PR, kendala).

        Cara pakai
            1. Kepala Kurikulum / Admin → Filament `/admin` menu **Akademik › Kurikulum**
                - Buat rencana kurikulum: pilih kelas + mapel + tahun ajaran + semester + guru
                - Tambah daftar topik per pekan di tab **Daftar Topik** (bab, tujuan, metode, media)
                - Klik action **🗓 Apply ke Tanggal** → isi rentang tanggal + hari aktif + jam →
                  sistem generate sesi otomatis dengan topik terdistribusi sequence
            2. Guru → Filament `/admin` menu **Akademik › Mengajar Hari Ini**
                - Lihat kartu sesi hari ini (urut by jam), klik **Mulai Mengajar** →
                  status jadi "Sedang Berlangsung"
                - Klik **Selesai & Catat** → isi pencapaian (%), catatan kegiatan, PR, kendala →
                  status jadi "Selesai"
                - Klik **Tampilkan Materi** untuk buka file materi di tab baru
            3. Admin → **Akademik › Sesi Mengajar** untuk manage semua sesi (filter, bulk publish/cancel)

        Notifikasi otomatis
            - Reminder WA + Email ke guru 30 menit sebelum jam mulai (via scheduler tiap 5 menit)
            - Template: "Reminder Mengajar: {mapel} - {kelas} ({jam}) — Topik: {topik}"

        Konfigurasi .env
            NOTIF_LESSON_REMINDER=true
            NOTIF_LESSON_MINUTES_BEFORE=30

        Tabel database
            - `curriculum_plans` (template): school_class_id, material_category_id, staff_member_id,
              academic_year, semester, title, default_methods, default_media
            - `curriculum_plan_topics` (topik per pekan): week_number, topic, learning_objectives,
              methods, media, assessment_plan, default_duration_minutes
            - `lesson_sessions` (instance per tanggal): session_date, start_time, end_time, topic,
              status (draft/published/ongoing/completed/cancelled), realisasi (achievement_percent,
              execution_notes, homework_notes, issues_notes)
            - `lesson_session_materials` (pivot ke ClassMaterial)
            - `lesson_session_assignments` (pivot ke ClassAssignment)

        Seeder demo
            - `CurriculumDemoSeeder` — 1 plan, 5 topik, 3 sesi hari ini
            - Jalankan: `php artisan db:seed --class=CurriculumDemoSeeder`
